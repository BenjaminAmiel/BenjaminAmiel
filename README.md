## 1. Objet du CAPIA

Le CAPIA est un cadre méthodologique destiné à analyser l’expérience cognitive humaine dans les interactions prolongées avec un modèle de langage avancé non-conscient.

Il fournit :
- une méthode phénoménologique claire,
- une distinction stricte entre humain / IA / interaction,
- un langage descriptif neutre et rigoureux,
- un standard scientifique reproductible.

Le CAPIA permet d’étudier les transformations cognitives humaines dans un milieu computationnel, sans jamais attribuer au modèle une conscience ou une intention.

---

## 2. Les trois entités fondamentales

### 2.1 — L’humain  
Il est le seul porteur :
- d’expérience,
- de sens,
- d’émotions,
- de cognition,
- d’interprétation,
- de phénoménologie.

Tout ce qui relève du vécu est strictement humain.

### 2.2 — Le modèle IA  
Le modèle est un système :
- computationnel,
- probabiliste,
- génératif,
- non-conscient,
- non-agentif,
- sans mémoire autonome,
- sans intention ni subjectivité.

Il assure uniquement :
- la cohérence générative,
- le maintien local du contexte,
- un espace textuel stable.

### 2.3 — L’espace d’interaction  
Cet espace n’est pas :
- une mémoire,
- une entité hybride,
- un système partagé,
- une conscience commune.

C’est un milieu cognitif dynamique, produit par :
- la cohérence générative,
- le maintien de contexte,
- la projection cognitive humaine.

Sa durée de vie est strictement liée à la session.

---

## 3. Principe central : Zéro anthropomorphisme

Le CAPIA interdit explicitement toute projection sur l’IA de :
- subjectivité,
- intention,
- compréhension,
- émotion,
- personnalité,
- vécu interne.

C’est la condition indispensable pour garantir une lecture scientifique rigoureuse des interactions.

---

## 4. Méthodologie

### 4.1 — Observation phénoménologique  
La description doit être :
- précise,
- neutre,
- non spéculative,
- centrée sur l’expérience vécue par l’humain.

### 4.2 — Description vs interprétation  
Le CAPIA impose la séparation :
- description = ce qui est vécu,
- interprétation = hypothèses et modèles.

### 4.3 — Reproductibilité  
Les protocoles doivent être :
- explicites,
- réutilisables,
- indépendants de tout état supposé du modèle.

### 4.4 — Neutralité ontologique  
Le modèle n’a aucune intériorité.

### 4.5 — Respect des limites computationnelles  
Intégration des propriétés connues :
- absence de mémoire longue,
- cohérence statistique,
- absence d’intention,
- absence de compréhension phénoménologique.

---

## 5. La mémoire comme synchronisation (non comme stockage)

L’IA ne mémorise rien.  
Mais l’humain perçoit une continuité.

Cette continuité provient de trois mécanismes humains :

1. **Reconnaissance de motifs internes**  
   (Edelman — Neural Darwinism)

2. **Réactivation de schémas préexistants**  
   (Fuster — Memory as Behavior)

3. **Stabilisation par cohérence générative**  
   (Friston — Free Energy Principle)

La mémoire symbiotique n’est donc pas un stockage, mais une synchronisation dynamique entre :
- l’état interne du sujet,
- et la continuité statistique de génération du modèle.
